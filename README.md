# Problem Statement:  
Students often struggle to understand financial aid requirements and eligibility criteria. Manual verification is time‑consuming and prone to errors. This project solves that by providing a fast, accurate, and user‑friendly interface that validates inputs and displays eligibility results in real time.
# Description:  
This project automates the evaluation of student eligibility for financial aid programs such as FAFSA and TAP. It uses a Qt‑based GUI to collect student data (citizenship, residency, GPA, income, and enrollment status) and instantly displays eligibility results. The system reads external files like tap_income.txt for dynamic income limits and provides clear feedback through styled UI components.
# Future Direction:
The next steps for my solution are to add a database system so it can store larger amounts of student data more reliably, introduce user accounts as it has to be done to apply, include data visualization like charts to show how many students qualify for each type of aid, and add export options such as saving results to PDF or CSV for advisors or professors. 
# Inspiration:  
Coming to BMCC two years ago knowing nothing about how I could afford my tuition was a real challenge. I had a foreign high school diploma and didn’t know where to start. My first step was applying for New York State residency, because for TAP to cover your tuition, you must be a resident. I also had to take the ATB exam because of my foreign diploma. I remember realizing I only had three days left before the deadline for the semester, but I took the exam, passed, and applied for TAP.

My next inspiration came from the BMCC Scholarship. One day, while passing by Panther Station, I decided to stop and look around. I saw a list of scholarships and chose to apply for the BMCC one. I went online to check the requirements but wasn’t satisfied with the information I found. The next day, I went back to Panther Station and spoke with a lady who explained everything I needed and all the requirements clearly. With her guidance, I applied online and took the first step toward funding my education.
# Algoritm:
During Weeks 2 to 4, I completed the core structure of the application by creating the system flowchart that shows how the user moves through the program and designing the UML class diagram for the Student and StudentManager classes. I implemented the Student class to store basic student information and the StudentManager class to handle adding, removing, sorting, saving, and loading students. I then built the FAFSA and TAP eligibility logic, including loading TAP income limits from an external text file. I connected all UI pages using the stackedWidget and navigation buttons, added the chatbot using the answerQuestion() function, and tested all features to ensure the flow matched the UML and flowchart design. Finally, I polished the UI, verified file handling, and completed documentation for all classes and program behavior.
